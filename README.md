# Транслятор для подмножества Markdown в HTML на ANTLR4

## Что такое ANTLR4

**ANTLR4** (ANother Tool for Language Recognition) — генератор лексеров и парсеров. По файлу с грамматикой (`.g4`) он генерирует C++ код, который читает текст, разбивает его на токены и строит дерево разбора.

В отличие от Bison/Flex, где лексер и парсер описываются в двух раздельных файлах (`.l` и `.y`), в ANTLR4 вся грамматика — в одном файле `.g4`. Парсер строит полное дерево разбора (CST) **автоматически**, а семантика реализуется отдельным обходом этого дерева.

## Подмножестов Markdown
1. Загаловки (с 1 по 6 уровень)
```
# Заголовок 1
## Заголовок 2
### Заголовок 3
```
HTML:
```
<h1>Заголовок 1</h1>
<h2>Заголовок 2</h2>
<h3>Заголовок 3</h3>
```
2. Параграфы
```
Это обычный текст.

Это новый параграф.
```
HTML:
```
<p>Это обычный текст.</p>
<p>Это новый параграф.</p>
```
3. Выделение текста (курсив/жирный)
```
*курсив*

**жирный**
```
HTML:
```
<i>курсив</i>

<b>жирный</b>
```
4. Списки
```
- Первый пункт
- Второй пункт
- Третий пункт
```
HTML:
```
<ul>
    <li>Первый пункт</li>
    <li>Второй пункт</li>
    <li>Третий пункт</li>
</ul>
```
5. Ссылки
```
[GitHub](https://github.com)
```
HTML:
```
<a href="https://github.com">GitHub</a>
```
6. Кодовые блоки/встроенный код
7. Картинки (имя файла из текущей директории или полный путь до файла)
```
![Изображение не найдено](src\image.png)
```
HTML:
```
<img src="src\image.png" alt="Alt-Изображение не найдено">
```
8. Чекбоксы
```
- [x] Выполнено
- [ ] Не выполнено
```
HTML:
```
<li><input type="checkbox" checked disabled> Выполнено</li>
<li><input type="checkbox" disabled> Не выполнено</li>
```
9. Цитаты
```
> Это цитата
```
HTML:
```
<blockquote>Это цитата</blockquote>
```

## Сборка и использование
```
cmake --build build
```
В папке ```./build/``` будет создан исполняемый файл ```main```.
Использование и вывод в командную строку:
```
./build/main test.md
```
Создать/перезаписать HTML файл:
```
./build/main test.md -o test.html
```
## Ошибки

Транслятор поддерживает обработку синтаксических и семантических ошибок.

Синтаксические ошибки

Возникают при нарушении структуры Markdown.

Примеры:

- незакрытый блок кода;
- некорректная ссылка;
- неправильное использование специальных символов.

Сообщения выводятся в stderr.

Пример:
```
line 5:10 mismatched input EOF
```
После синтаксической ошибки программа завершает работу с ненулевым кодом возврата.

Семантические ошибки

Возникают при некорректной разметке, которая формально может быть разобрана, но содержит логические ошибки.

Формат сообщения:
```
Semantic error [line N]: <message>
```
Пример:
```
Semantic error [line 8]: empty link URL
```
